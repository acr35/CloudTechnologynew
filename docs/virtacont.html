Virtualization and Containerization

Virtualization

Virtualization is basically the emulation of a container system which means that it makes it possible to run what appears to be on many separate computers on hardware that is actually one computer. The operating systems and their applications share hardware resources from a single host server or from a pod of host servers

  • running multiple applications on the same physical hardware
  • placing each application on its own virtual machine

Pros

  • all Os resources are to apps
  • established security and management tools
  • better known security controls

Cons

  • virtual machines bulky in size
  • running multiple virtual machines lead to unstable performance
  • boot up process take longe time and virtual machines not solve the problem

Containerization

Instead of virtualizing the underlying computer, only the operating system virtualized. Container sit on top of a physical server and each container shares the host operating system kernel and usually the binaries. Server can run multiple workloads with a single operating system installation . Also, containers can create portable consistent operating environments for development testing.

Similarities and Differences Between Virtualization and Containerization:

  • while containers are exceptionally light and take few seconds to start, virtual machines take minutes to run and are an order of magnitude larger than the equivalent container.
  • virtualization create many Operating systems in one host machine, while containerization create multiple containers for each type of application.
  • you can put two to three as many applications in a single server with containers that you can with virtual machines
